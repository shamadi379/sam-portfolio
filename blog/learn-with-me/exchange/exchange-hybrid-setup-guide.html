Exchange Hybrid Setup

Step-by-step guide to configure Exchange hybrid deployment

Oct 7, 2025 8 min read

Exchange hybrid allows seamless coexistence between on-premises Exchange and Exchange Online. Perfect for gradual migrations or keeping some mailboxes on-premises while leveraging cloud features.

Prerequisites Check

‚úÖ On-Premises Requirements

  • ‚Ä¢ Exchange 2016+ (Latest CU)
  • ‚Ä¢ Valid public SSL certificate
  • ‚Ä¢ Internet connectivity on port 443
  • ‚Ä¢ Public DNS records configured

‚úÖ Office 365 Requirements

  • ‚Ä¢ Exchange Online Plan 1/2
  • ‚Ä¢ Verified domain in tenant
  • ‚Ä¢ Azure AD Connect configured
  • ‚Ä¢ Global Admin permissions

1 Prepare On-Premises Environment 

# Check Exchange prerequisites for hybrid
$exchangeServer = Get-ExchangeServer | Where-Object {$_.ServerRole -like "*ClientAccess*"}

# Verify Exchange version
Write-Output "Exchange Version: $($exchangeServer.AdminDisplayVersion)"

# Check SSL certificate
$cert = Get-ExchangeCertificate | Where-Object {$_.Services -like "*IIS*"} | 
    Sort-Object NotAfter -Descending | Select-Object -First 1

Write-Output "SSL Certificate Expires: $($cert.NotAfter)"
Write-Output "Certificate Subject: $($cert.Subject)"

# Test connectivity to Office 365
Test-NetConnection -ComputerName outlook.office365.com -Port 443

# Verify accepted domains
Get-AcceptedDomain | Select-Object Name, DomainName, DomainType

2 Configure DNS Records

Required DNS Records

Record Type Name Value
A mail.company.com Public IP of Exchange
CNAME autodiscover.company.com mail.company.com

# Test DNS resolution
$domain = "company.com"

# Test Autodiscover
$autodiscover = Resolve-DnsName -Name "autodiscover.$domain" -ErrorAction SilentlyContinue
if ($autodiscover) {
    Write-Output "‚úì Autodiscover DNS record found: $($autodiscover.NameHost)"
} else {
    Write-Warning "‚úó Autodiscover DNS record missing"

# Test MX record

if ($mx) {
    Write-Output "‚úì MX record found: $($mx.NameExchange)"

    Write-Warning "‚úó MX record missing"

3 Run Hybrid Configuration Wizard

Download & Install HCW

  1. 1. Download from aka.ms/hybridwizard
  2. 2. Run as Administrator on Exchange server
  3. 3. Sign in with Global Admin account
  4. 4. Select "Full Hybrid Configuration"
# Verify hybrid configuration after HCW
Get-HybridConfiguration | Select-Object *

# Check organization relationship
Get-OrganizationRelationship | Where-Object {$_.TargetApplicationUri -like "*outlook.com"} | 
    Select-Object Name, Enabled, DomainNames

# Verify send/receive connectors
Get-SendConnector | Where-Object {$_.AddressSpaces -like "*mail.onmicrosoft.com*"}
Get-ReceiveConnector | Where-Object {$_.RemoteIPRanges -like "*outlook.com*"}

# Test mail flow
Test-MigrationServerAvailability -ExchangeOutlookAnywhere -Autodiscover `
    -EmailAddress "user@company.onmicrosoft.com" -Credentials (Get-Credential)

4 Test Mail Flow & Features 

# Test mail flow between on-premises and cloud
# From Exchange Management Shell

# Test outbound mail flow (on-prem to cloud)
$cloudUser = "clouduser@company.onmicrosoft.com"
$onPremUser = "onpremuser@company.com"

# Send test email from on-premises to cloud
Send-MailMessage -From $onPremUser -To $cloudUser -Subject "Hybrid Test - OnPrem to Cloud" `
    -Body "Testing mail flow from on-premises to Exchange Online" -SmtpServer "mail.company.com"

# Test inbound mail flow (cloud to on-prem)
# Run from Exchange Online PowerShell
Connect-ExchangeOnline

Send-MailMessage -From $cloudUser -To $onPremUser -Subject "Hybrid Test - Cloud to OnPrem" `
    -Body "Testing mail flow from Exchange Online to on-premises"

# Check message trace
Get-MessageTrace -SenderAddress $onPremUser -RecipientAddress $cloudUser -StartDate (Get-Date).AddHours(-1)

# Test free/busy
Get-MailboxCalendarFolder -Identity "$cloudUser:\Calendar" | 
    Set-MailboxCalendarFolder -PublishEnabled $true -DetailLevel AvailabilityOnly

5 Migrate Mailboxes & Ongoing Management 

# Create migration batch to move mailboxes to cloud
$users = @("user1@company.com", "user2@company.com")
$migrationEndpoint = Get-MigrationEndpoint | Where-Object {$_.EndpointType -eq "ExchangeOutlookAnywhere"}

# Create CSV for batch migration
$users | ForEach-Object { [PSCustomObject]@{EmailAddress = $_} } | 
    Export-Csv -Path ".\HybridMigration.csv" -NoTypeInformation

# Create migration batch
New-MigrationBatch -Name "HybridBatch1" -CSVData ([System.IO.File]::ReadAllBytes(".\HybridMigration.csv")) `
    -TargetDeliveryDomain "company.mail.onmicrosoft.com" -AutoStart

# Monitor migration progress
Get-MigrationBatch "HybridBatch1" | Select-Object Status, TotalCount, SyncedItemCount

# Move mailbox back to on-premises (if needed)
New-MoveRequest -Identity "user@company.com" -Remote -RemoteHostName "mail.company.com" `
    -RemoteCredential (Get-Credential) -TargetDeliveryDomain "company.com"

Common Issues & Solutions

üö® Autodiscover Issues

Problem: Outlook can't connect to Exchange Online

Solution: Verify SCP and external URL settings

# Fix Autodiscover configuration
Set-ClientAccessService -Identity "ExchangeServer" -AutoDiscoverServiceInternalUri "https://mail.company.com/Autodiscover/Autodiscover.xml"
Get-ClientAccessService | Set-ClientAccessService -AutoDiscoverServiceInternalUri $null

‚ö†Ô∏è Free/Busy Not Working

Problem: Calendar free/busy not showing between environments

Solution: Check organization relationships

# Fix free/busy sharing
Get-OrganizationRelationship | Set-OrganizationRelationship -FreeBusyAccessEnabled $true -FreeBusyAccessLevel LimitedDetails
